  • 재무정보 및 주소 작성
  • BPM 승인 처리

  • 프로그램 실행 및 기본정보 입력
  • 청구내용 입력 (송장처리일, 금액, 세액)
  • 매출정보 입력 (G/L계정, 코스트센터)
  • 결재요청 및 BPM 승인

🧾ERP 청구서 발행

  • 발행정보 조회

👥 고객처 등록 (ZFIRE4090)

1프로그램 실행

SAP에서 고객 마스터를 생성하거나 변경하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4090 입력
  • Enter 키를 눌러 프로그램 실행
  • 작업 유형을 선택한 후 요청자 입력

주의사항: t-code는 정확히 입력해야 합니다.

2신청내역 작성

고객 등록을 위한 신청 사유를 입력합니다.

  • 신청 사유를 구체적으로 작성
  • 등록 목적과 배경 설명
  • 필요시 추가 설명 기재

팁: 신청사유는 구체적으로 작성하면 승인이 빨라집니다.

3기본정보 작성

고객의 기본 정보를 입력합니다.

  • 계정그룹: 고객 유형에 맞는 계정그룹 선택
  • 고객명: 정식 회사명 또는 고객명 입력
  • 고객단축명: 시스템에서 사용할 단축명 입력

주의: 고객명은 사업자등록증과 일치해야 합니다.

4세무정보 작성

세무 처리를 위한 정보를 입력합니다.

  • 사업자등록번호: 사업자등록증 상의 번호
  • 대표자명: 사업자등록증 상의 대표자
  • 업태: 사업자등록증 상의 업태
  • 업종: 사업자등록증 상의 업종

필수서류: 사업자등록증을 미리 준비해주세요.

입력 완료 후 BPM결재요청 버튼을 클릭합니다.

5재무정보 작성

결제 관련 정보를 입력합니다.

  • 지급조건: 결제 조건 선택
  • 가상계좌은행: 신한은행(088) 지정

참고: 가상계좌는 시스템에서 자동 생성됩니다.

6주소 기재

고객의 주소 및 연락처 정보를 입력합니다.

  • 주소 검색: 우편번호 검색으로 정확한 주소 입력
  • 연락처 정보:
    • 전화번호
    • E-mail 주소
  • 청구지 주소: 주소 검색으로 청구서 발송 주소 입력

모든 정보 입력 완료 후 BPM 요청을 진행합니다.

7BPM 승인

📋 BPM 승인 절차

  • 1단계: 입력정보 확인
  • 2단계: 증빙서류 첨부
    • 사업자등록증
    • 개인정보 수집이용동의서
    • 법인인감증명서
    • 구매처 등록서류 (해당시)
  • 3단계: 업무처리 클릭하여 승인 요청

승인 경로: 재무회계팀에서 최종 승인 처리됩니다.

고객 마스터 생성/변경 프로그램 ZFIRE4090

📝 수입전표 입력 (ZFIRE4110)

1프로그램 실행

수입전표를 입력하여 매출을 등록하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4110 입력
  • Enter 키를 눌러 프로그램 실행
  • 고객사업자번호 입력
  • 전표헤더 텍스트 한줄 작성

주의: 고객사업자번호는 정확히 입력해야 합니다.

2청구방식 설정

청구 유형과 채권 유형을 선택합니다.

  • 개별항목 선택:
    • 매월청구수입전표-정기
    • 1회 청구수입전표-수기
  • 채권유형:
    • 기본: 영업수익
    • 영업수익이 아닐 경우: "영업외수익" 선택

설정 완료 후 실행 버튼을 클릭합니다.

참고: 선수금은 별도 문의 필요합니다.

3청구내용 입력

세금계산서 발행을 위한 상세 정보를 입력합니다.

  • 송장 처리일: 세금계산서 발행일(작성일) 입력
    • ⚠️ 입력일 기준 전월 일자 입력 금지
  • 금액: VAT 포함 금액 입력
  • 세액: VAT 입력
    • 면세일 경우 우측 세금코드 변경
  • 텍스트: 청구 내용 상세 설명

4매출정보 입력

회계 처리를 위한 상세 정보를 입력합니다.

  • G/L계정: 적정계정 입력
    • 문의: 경영재무팀 (예산은 경영성과팀)
  • 전표 통화 표시금액: 매출금액(VAT제외)
  • 코스트센터: 예산 매핑된 코스트센터
  • WBS: 필요시 입력
  • 수익성세그먼트: 관련 BM에 맞는 세그먼트

입력 완료 후 시뮬레이션으로 확인 → 완료 상태로 저장

5결재요청

ZFIRE0092 프로그램을 통해 결재를 요청합니다.

📋 결재요청 절차

  • 1단계: ZFIRE0092 실행
  • 2단계: 전표입력 일자 범위 설정
  • 3단계: 실행 버튼 클릭
  • 4단계: 상세적요 입력 버튼 클릭
  • 5단계: 가산금 부과 정보 입력 버튼 클릭 (해당시)
  • 6단계: BPM 결재요청

승인: 전결기준에 따라 적정 증빙 첨부하여 결재

수입전표 입력 ZFIRE4110

🧾 ERP 청구서 발행 (ZFIRE4130)

1프로그램 실행

발생된 수익에 대한 청구서를 발행하고 관리하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4130 입력
  • Enter 키를 눌러 프로그램 실행
  • 고객코드 입력
  • 실행버튼 클릭하여 청구서 정보 조회

기능: 발행정보 조회, 청구서 재발송, PDF 다운로드 가능

2발행정보 조회

고객별 청구서 발행 현황을 조회할 수 있습니다.

  • 조회 정보:
    • 전표번호
    • 청구 금액
    • 발행일자
    • 청구서 상태
  • 필터 옵션:
    • 발행기간 설정
    • 청구서 상태별 조회
    • 금액별 정렬

활용: 청구서 발행 이력 관리 및 상태 확인

3청구서 재발송

이미 발행된 청구서를 다시 고객에게 전송할 수 있습니다.

  • 재발송 절차:
    • 발행된 청구서 목록에서 선택
    • "이메일청구서발송" 버튼 클릭
    • 수신 이메일 주소 확인
    • 발송 버튼 클릭
  • 주의사항:
    • 고객 정보의 이메일 주소가 정확한지 확인
    • 중복 발송 주의

팁: 발송 전 이메일 주소를 다시 한번 확인해주세요.

4PDF 다운로드

청구서를 PDF 파일로 다운로드하여 보관하거나 별도 전송할 수 있습니다.

  • 다운로드 절차:
    • 청구서 목록에서 원하는 항목 선택
    • "청구서 발행" 버튼 클릭
    • "PDF파일 다운로드" 버튼 클릭
    • 파일 저장 위치 선택
  • 활용 방법:
    • 내부 보고용 자료
    • 고객 요청시 별도 전송
    • 회계 증빙 보관

참고: PDF 파일은 정식 청구서와 동일한 효력을 가집니다.
