📈매출 전표 개요

🎯 매출 전표 처리 프로세스

매출 전표는 회사의 수익을 인식하고 고객에게 청구서를 발행하는 과정입니다.

  • 1단계: 고객처 등록 - 새로운 고객 정보를 시스템에 등록
  • 2단계: 수입전표 입력 - 매출 발생에 대한 전표 작성
  • 3단계: ERP 청구서 발행 - 고객에게 청구서 발송
💡 주요 특징
• 자동 세금계산서 발행
• 실시간 매출 인식
• 통합 고객 관리
• BPM 승인 워크플로우

👥고객처 등록 (ZFIRE4090)

🔧 프로그램 실행

SAP에서 고객 마스터를 생성하거나 변경하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4090 입력
  • Enter 키를 눌러 프로그램 실행
  • 작업 유형을 선택한 후 요청자 입력
고객 마스터 생성/변경 프로그램 ZFIRE4090

📝 기본정보 작성

  • 계정그룹: 고객 유형에 맞는 계정그룹 선택
  • 고객명: 정식 회사명 또는 고객명 입력
  • 고객단축명: 시스템에서 사용할 단축명 입력
⚠️ 주의
고객명은 사업자등록증과 일치해야 합니다.

🏛️ 세무정보 작성

  • 사업자등록번호: 사업자등록증 상의 번호
  • 대표자명: 사업자등록증 상의 대표자
  • 업태: 사업자등록증 상의 업태
  • 업종: 사업자등록증 상의 업종

📋 BPM 승인 절차

  • 1단계: 입력정보 확인
  • 2단계: 증빙서류 첨부 (사업자등록증, 개인정보 수집이용동의서 등)
  • 3단계: 업무처리 클릭하여 승인 요청

📝수입전표 입력 (ZFIRE4110)

🚀 프로그램 실행

수입전표를 입력하여 매출을 등록하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4110 입력
  • Enter 키를 눌러 프로그램 실행
  • 고객사업자번호 입력
  • 전표헤더 텍스트 한줄 작성

⚙️ 청구방식 설정

청구 유형과 채권 유형을 선택합니다.

  • 개별항목 선택:
    • 매월청구수입전표-정기
    • 1회 청구수입전표-수기
  • 채권유형:
    • 기본: 영업수익
    • 영업수익이 아닐 경우: "영업외수익" 선택
💡 선수금은 별도 문의 필요합니다.

💰 청구내용 입력

  • 송장 처리일: 세금계산서 발행일(작성일) 입력
  • 금액: VAT 포함 금액 입력
  • 세액: VAT 입력 (면세일 경우 우측 세금코드 변경)
  • 텍스트: 청구 내용 상세 설명
⚠️ 중요
입력일 기준 전월 일자 입력 금지

📊 매출정보 입력

  • G/L계정: 적정계정 입력 (문의: 경영재무팀)
  • 전표 통화 표시금액: 매출금액(VAT제외)
  • 코스트센터: 예산 매핑된 코스트센터
  • WBS: 필요시 입력
  • 수익성세그먼트: 관련 BM에 맞는 세그먼트

🧾ERP 청구서 발행 (ZFIRE4130)

🚀 프로그램 실행

발생된 수익에 대한 청구서를 발행하고 관리하는 프로그램입니다.

  • SAP 왼쪽 상단에 t-code ZFIRE4130 입력
  • Enter 키를 눌러 프로그램 실행
  • 고객코드 입력
  • 실행버튼 클릭하여 청구서 정보 조회

📊 발행정보 조회

고객별 청구서 발행 현황을 조회할 수 있습니다.

  • 조회 정보: 전표번호, 청구 금액, 발행일자, 청구서 상태
  • 필터 옵션: 발행기간 설정, 청구서 상태별 조회, 금액별 정렬

📧 청구서 재발송

이미 발행된 청구서를 다시 고객에게 전송할 수 있습니다.

  • 발행된 청구서 목록에서 선택
  • "이메일청구서발송" 버튼 클릭
  • 수신 이메일 주소 확인
  • 발송 버튼 클릭

📄 PDF 다운로드

청구서를 PDF 파일로 다운로드하여 보관하거나 별도 전송할 수 있습니다.

  • 청구서 목록에서 원하는 항목 선택
  • "청구서 발행" 버튼 클릭
  • "PDF파일 다운로드" 버튼 클릭
  • 파일 저장 위치 선택
💡 PDF 파일은 정식 청구서와 동일한 효력을 가집니다.

📉비용 전표 개요

🎯 비용 전표 처리 프로세스

비용 전표는 회사에서 발생하는 각종 지출을 처리하고 관리하는 과정입니다.

  • 1단계: 구매처 등록 - 새로운 공급업체 정보를 시스템에 등록
  • 2단계: 지출전표 입력 - 비용 발생에 대한 전표 작성
  • 3단계: BPM 결재 요청 - 승인 워크플로우 진행
💡 주요 특징
• 세금계산서 연동 처리
• 다양한 세금코드 지원
• 외화 지출 처리
• 선급금 관리

🏪구매처 등록

🔧 국내 구매처 등록

국내 공급업체를 시스템에 등록하는 과정입니다.

  • 구매처 마스터 생성 메뉴 접속
  • 사업자등록번호 입력
  • 업체명, 대표자명 입력
  • 주소, 연락처 정보 입력
  • 은행계좌 정보 등록
⚠️ 필수 서류
사업자등록증, 통장사본, 법인인감증명서

🌐 외화 구매처 등록 (ZFIRR6100)

해외 공급업체를 등록할 때 사용하는 프로세스입니다.

  • 계정그룹: 해당하는 사업자 유형 선택
  • 국가코드: 해당 국가 선택
  • 사업자유형: 12(외국기업) 선택
  • 은행정보:
    • 국가코드: KR 외 선택
    • 은행코드: 997 (해외은행) 선택
    • 계좌번호: 인보이스 등에 기재된 계좌번호
💡 해당 서류가 없을 경우 담당부서에 대체가능한 서류 문의 (예: 인보이스 등)

📋지출전표 입력 (세금계산서)

🔍 전자세금계산서 조회 (ZEVRR0040)

먼저 전자세금계산서가 있는지 확인합니다.

  • 사업자등록번호 또는 공급받는자 이메일 입력
  • 조회 버튼 클릭하여 세금계산서 확인
  • 전표 미작성 건에 대해 전표 발행 진행
💡 증빙(세금계산서, 계산서)가 없는 경우, 바로 임시전표 생성(FV60)부터 진행

📝 지급 전표 생성 (FV60)

T-Code: FV60 - AP 임시진표 입력

🔸 기본 정보 입력

  • 공급업체: 구매처 마스터에 등록된 공급업체 번호
  • 송장 처리일: 세금계산서/계산서 발행일
  • 전기일: 장부 반영일
  • 금액: 총 금액 (부가세 포함)
  • 세액: 부가세 금액
  • 사업장: 0000

🔸 세금코드 선택

J2

세금계산서가 있는 일반 과세 비용

L4

세금계산서가 있는 접대비, 업무추진비
(세액 금액을 0원으로 처리)

R1

계산서가 있는 면세 비용

Z5

세금계산서, 계산서가 없는 비용

🔸 비용 계정 및 센터 정보

  • G/L 계정: 비용 계정과목 선택
  • 코스트센터: 기안자 소속 부서
  • 손익센터: 코스트센터 입력 후 자동 설정
  • 텍스트: 비용항목 적요 내역
  • 수익성세그먼트: 직접비 사용시 BM(L4) 필수

💰 지급 조건 설정

📅 대가 지급일 기준

세금계산서 발행일 대가 지급일
1~10일 당월 24일
11~20일 익월 4일
21~말일 익월 14일
  • 지급조건: 1000 (현금 지급 기준)
  • 파트너은행: 구매처 지급 은행 선택
⚠️ 중요
세금계산서 발행일로부터 7일 이내 전표처리 완료 필요
지연 시 지출 결의 +7일 이후 도래하는 대가 지급일 적용

✅ 시뮬레이션 및 저장

  • 시뮬레이션 클릭하여 작성내용 검토
  • 완료 상태로 저장 선택
  • 하단 계좌 정보 확인 후 엔터
  • 임시저장 완료 확인

📨 BPM 결재 요청 (ZFIRE0092)

T-Code: ZFIRE0092 - 결재 요청

  • 회사코드: 1000
  • 기안 기간: 조회할 기간 입력 후 실행
  • 결재 요청할 전표 선택
  • 상세 적요 입력
  • BPM 결재 요청 버튼 클릭

📋 BPM 기안 작성

  • 첨부파일: 기안 문서에 필요한 증빙 자료 첨부
  • 결재선 지정: 지출 전결기준에 따른 결재선 지정
  • 물품구매여부: 해당없음/선장품/업무용 용품 중 선택
  • 업무처리: BPM 기안 상신

📄지출전표 입력 (세금계산서 없음)

📝 임시전표 생성 (FV60)

세금계산서나 계산서가 없는 경우의 전표 처리입니다.

  • 세금코드: Z5 (세금계산서, 계산서가 없는 비용)
  • 적용 케이스:
    • 교통비, 숙박비
    • 회의비, 식대
    • 기타 간이영수증 등
⚠️ 주의사항
세금계산서가 없는 경우에도 적절한 증빙서류(영수증, 카드매출전표 등)가 필요합니다.

📋 전표 조회 및 수정 (FBV2)

T-Code: FBV2 - 임시 전표 조회

📋 방법 1: 전표 번호를 알고 있는 경우

  • 전표 번호 직접 입력
  • 상단 체크박스 클릭하여 조회

📋 방법 2: 전표 번호를 모르는 경우

  • 문서 리스트 조회 선택
  • 검색조건 설정 후 조회 (미설정시 전체 조회)
  • 조회할 전표 더블 클릭
💡 BPM 결재 상신 전까지만 수정 가능
결재 상신 후에는 BPM 취소 후 수정 필요

🔄 합산전표 생성 (ZTRRE0150)

여러 개의 전표를 하나의 기안문으로 작성하고 싶을 때 사용

  • 조회할 전표의 전기일 입력
  • 전표작성자 사번 입력
  • 실행하여 전표 목록 조회
  • 합산할 전표들 선택
  • 합산처리 및 적요 입력
  • 합산전표 확인 후 BPM 결재요청

💱외화 지출전표 입력

🔧 지출전표 작성 (FV60)

외화로 지급하는 비용에 대한 전표 처리입니다.

  • 공급업체: 지급 대상업체 입력
  • 송장 처리일 = 전기일: 세금계산서 발행일
  • 지급통화로 변경: USD, JPY 등
  • 금액: 지급통화에 맞게 작성 (원화 환산금액 X)
  • 세금코드: Z5
  • 텍스트: 지급목적, 내용 등 작성
⚠️ 외화 지급 특이사항
• 외화는 매월 말 지급처리
• 지급방법: D
• 지급조건: 9000번 사용

📊 세부내용 작성

  • G/L계정: 안내받은 계정 기재
  • 전표 통화 표시 금액: 외화 금액과 동일
  • 텍스트, 코스트센터, 수익성세그먼트: 기재

🏦 지급 설정

  • 지급조건: 9000번, 만기일이 해당 달의 말일이 될 수 있도록 일수 수정
  • 지급방법: D
  • 파트너은행: 지정 (칸 오른편 버튼 클릭하여 은행 지정)

📨 지출결의 상신 (ZFIRE0092)

  • 대상 전표 선택
  • 상세적요 입력 클릭
  • 지급대상, 목적, 금액 등 세부내용 작성
  • BPM 결재요청 클릭
💡 결재선: 원화환산금액 기준 지출전결과 동일
인보이스, 계약서 등 증빙서류 첨부 후 결재상신

💰선급금 전표 입력

🔧 선급금 전표 작성 (F-43)

미리 지급하는 금액에 대한 전표 처리입니다.

  • T-Code: F-43 입력 (선지급시 선급금 전표 작성)
  • G/L계정: 선급금_기타 (114018000)
  • 지급조건: 1000 또는 1500 입력 (지급기산일 입력 X)
💡 선급금은 나중에 실제 비용 발생시 정산처리됩니다.

📋 세금계산서 연동

세금계산서 발행 후 매핑하여 전표 작성하는 경우

  • T-Code: ZEVRR0040 (세금계산서 조회)
  • 세금계산서 발행 후 세금계산서 매핑하여 전표 작성
  • 전표 발행 진행

⚙️ 파트너은행 설정

  • 파트너은행 입력은 상단 추가데이터 메뉴에서 입력
  • 각 행을 선택하면 수정 화면으로 넘어감
  • 수정화면에서 다시 뒤로가기 버튼을 누르면 이전 화면으로

💾 저장 및 승인

  • 전표 메뉴에서 완료상태로 저장
  • ZFIRE0092 T-CODE 입력하여 BPM 승인

📚용어 설명

Tt-code

정의: SAP 시스템에서 특정 기능/화면으로 바로 이동하기 위해 사용하는 단축코드

  • 사용방법: SAP 화면 좌측상단 박스에 입력 후 Enter
  • 예시: ZFIRE4090, ZFIRE4110, ZFIRE4130
  • 특징: 각 t-code는 고유한 기능을 가지고 있음
  • 장점: 메뉴를 통해 이동하는 것보다 빠르고 효율적

GG/L계정

정의: SAP 시스템에 입력되어 있는 회계 계정으로, 총계정원장(General Ledger)의 약자

  • 용도: 회계 거래의 분류 및 기록
  • 구성: 숫자로 이루어진 계정 코드
  • 예시: 매출계정, 비용계정, 자산계정 등
  • 문의처: 경영재무팀 (예산 관련은 경영성과팀)

C코스트센터

정의: 손익계정 구분을 위해 부여한 부서 고유번호

  • 목적: 예산관리와 BM별 손익구분
  • 특징: 각 부서마다 고유한 번호 할당
  • 활용: 비용 배분 및 성과 측정
  • 필수성: 예산 관리를 위해 필수적인 정보

P수익성세그먼트

정의: 회계 계정보다 상세한 BM정보를 구분하기 위해 입력하는 정보

  • 상세도: BM L5까지 구분됨
  • 용도: 세밀한 수익성 분석
  • 활용: 사업부문별 성과 측정
  • 연계: 관련 BM에 맞는 세그먼트 선택 필요

WWBS

정의: 프로젝트에 해당되는 별도 예산관리를 위해 부여되는 고유번호

  • 풀네임: Work Breakdown Structure
  • 용도: 프로젝트별 예산 및 비용 관리
  • 특징: 각 프로젝트마다 고유한 WBS 할당
  • 입력시점: 프로젝트 관련 거래시에만 필요

P전기일

정의: 회계상 인식되는 일자

  • 용도: 회계 기간 구분 및 재무제표 작성
  • 특징: 실제 거래일과 다를 수 있음
  • 영향: 손익 인식 시점 결정
  • 관리: 회계 정책에 따라 설정

D증빙일

정의: 세무상 인식되는 일자로, 세금계산서 상 작성일과 동일

  • 기준: 세금계산서 작성일
  • 용도: 세무신고 및 부가세 신고
  • 중요성: 정확한 세무 처리를 위해 필수
  • 주의사항: 전기일과 구분하여 관리

❓자주 묻는 질문 (FAQ)

❓ "가용 예산이 부족하다고 뜹니다."

해결방법: 예산은 경영성과팀 또는 본부 내 재원담당자에게 문의하시기 바랍니다.

❓ "CO 오브젝트가 필요합니다."

해결방법: 코스트센터 입력이 누락된 경우입니다. 코스트센터를 입력하면 해결됩니다.

❓ "특정기간의 발생 손익의 경우 선급비용/선수수익계정을 사용하시기 바랍니다"

해결방법: 장기간에 걸쳐 반영되어야 할 비용을 일시에 지출하는 경우, 선급비용 전표 생성이 필요합니다.

일시에 반영되어야 할 비용이면 [ENTER] 후 넘어가시면 됩니다.

💡 예시: 1년치 임차비, 6개월 용역 대금 등을 하나의 세금계산서로 수령하여 일시에 지출하는 경우

❓ "전기는 제로잔액으로만 가능합니다; 전표를 수정하십시오"

해결방법: FV60에서 금액은 "공급가액+세액=합계" 금액입니다.

전표 표시 통화금액(공급가액) + 세액 = 금액(합계)가 되도록 전표 수정이 필요합니다.

❓ BPM 결재가 반려되었습니다. 어떻게 해야 하나요?

해결방법:

  1. 먼저 BPM에서 취소 처리를 진행합니다
  2. SAP에서 임시전표를 수정합니다
  3. 수정 완료 후 다시 BPM 결재 요청을 진행합니다
⚠️ BPM 취소 처리를 먼저 진행하지 않으면 임시전표 수정이 불가능합니다.

❓ 여러 전표를 하나의 결재로 올리고 싶습니다.

해결방법: 합산전표 생성 기능(ZTRRE0150)을 사용하세요.

  • 동일한 날짜에 작성된 여러 전표를 하나로 합산 가능
  • 합산 후 하나의 BPM 기안문으로 결재 요청
  • 결재 효율성 향상 및 업무 간소화 효과

❓ 외화 지급은 어떻게 처리하나요?

해결방법: 외화 지급 전용 프로세스를 따라주세요.

  • 지급통화를 USD, JPY 등으로 변경
  • 세금코드는 Z5 사용
  • 지급조건은 9000번 사용
  • 매월 말 지급처리됩니다

❓ 세금코드를 어떻게 선택해야 하나요?

해결방법: 증빙 유형에 따라 다음과 같이 선택하세요.

J2

세금계산서가 있는 일반 과세 비용

L4

접대비, 업무추진비
(세액을 0원으로 처리)

R1

계산서가 있는 면세 비용

Z5

증빙서류가 없는 비용

❓ SAP 접속이 안 됩니다.

해결방법: 다음 단계를 확인해주세요.

  • 1단계: SAP가 설치되어 있는지 확인
  • 2단계: 연결 설정 확인
    • 어플리케이션 서버: 10.221.43.48
    • 인스턴스 번호: 00
    • 시스템 ID: KCP
  • 3단계: 로그인 정보 확인 (사번 7자리, 개인 비밀번호)
  • 4단계: 문제 지속시 IT 지원팀 문의

❓ 전표 번호를 잊어버렸습니다.

해결방법: FBV2에서 문서 리스트 조회를 사용하세요.

  • T-Code: FBV2 입력
  • "문서 리스트 조회" 선택
  • 검색조건 설정 (작성일, 금액 등)
  • 실행하여 전표 목록 확인
  • 해당 전표 더블클릭하여 상세 조회
