📚용어 설명

Tt-code

정의: SAP 시스템에서 특정 기능/화면으로 바로 이동하기 위해 사용하는 단축코드

  • 사용방법: SAP 화면 좌측상단 박스에 입력 후 Enter
  • 예시: ZFIRE4090, ZFIRE4110, ZFIRE4130
  • 특징: 각 t-code는 고유한 기능을 가지고 있음
  • 장점: 메뉴를 통해 이동하는 것보다 빠르고 효율적

GG/L계정

정의: SAP 시스템에 입력되어 있는 회계 계정으로, 총계정원장(General Ledger)의 약자

  • 용도: 회계 거래의 분류 및 기록
  • 구성: 숫자로 이루어진 계정 코드
  • 예시: 매출계정, 비용계정, 자산계정 등
  • 문의처: 경영재무팀 (예산 관련은 경영성과팀)

C코스트센터

정의: 손익계정 구분을 위해 부여한 부서 고유번호

  • 목적: 예산관리와 BM별 손익구분
  • 특징: 각 부서마다 고유한 번호 할당
  • 활용: 비용 배분 및 성과 측정
  • 필수성: 예산 관리를 위해 필수적인 정보

P수익성세그먼트

정의: 회계 계정보다 상세한 BM정보를 구분하기 위해 입력하는 정보

  • 상세도: BM L5까지 구분됨
  • 용도: 세밀한 수익성 분석
  • 활용: 사업부문별 성과 측정
  • 연계: 관련 BM에 맞는 세그먼트 선택 필요

WWBS

정의: 프로젝트에 해당되는 별도 예산관리를 위해 부여되는 고유번호

  • 풀네임: Work Breakdown Structure
  • 용도: 프로젝트별 예산 및 비용 관리
  • 특징: 각 프로젝트마다 고유한 WBS 할당
  • 입력시점: 프로젝트 관련 거래시에만 필요

P전기일

정의: 회계상 인식되는 일자

  • 용도: 회계 기간 구분 및 재무제표 작성
  • 특징: 실제 거래일과 다를 수 있음
  • 영향: 손익 인식 시점 결정
  • 관리: 회계 정책에 따라 설정

D증빙일

정의: 세무상 인식되는 일자로, 세금계산서 상 작성일과 동일

  • 기준: 세금계산서 작성일
  • 용도: 세무신고 및 부가세 신고
  • 중요성: 정확한 세무 처리를 위해 필수
  • 주의사항: 전기일과 구분하여 관리

❓자주 묻는 질문 (FAQ)

❓ "가용 예산이 부족하다고 뜹니다."

해결방법: 예산은 경영성과팀 또는 본부 내 재원담당자에게 문의하시기 바랍니다.

❓ "CO 오브젝트가 필요합니다."

해결방법: 코스트센터 입력이 누락된 경우입니다. 코스트센터를 입력하면 해결됩니다.

❓ "특정기간의 발생 손익의 경우 선급비용/선수수익계정을 사용하시기 바랍니다"

해결방법: 장기간에 걸쳐 반영되어야 할 비용을 일시에 지출하는 경우, 선급비용 전표 생성이 필요합니다.

일시에 반영되어야 할 비용이면 [ENTER] 후 넘어가시면 됩니다.

💡 예시: 1년치 임차비, 6개월 용역 대금 등을 하나의 세금계산서로 수령하여 일시에 지출하는 경우

❓ "전기는 제로잔액으로만 가능합니다; 전표를 수정하십시오"

해결방법: FV60에서 금액은 "공급가액+세액=합계" 금액입니다.

전표 표시 통화금액(공급가액) + 세액 = 금액(합계)가 되도록 전표 수정이 필요합니다.

❓ BPM 결재가 반려되었습니다. 어떻게 해야 하나요?

해결방법:

  1. 먼저 BPM에서 취소 처리를 진행합니다
  2. SAP에서 임시전표를 수정합니다
  3. 수정 완료 후 다시 BPM 결재 요청을 진행합니다
⚠️ BPM 취소 처리를 먼저 진행하지 않으면 임시전표 수정이 불가능합니다.

❓ 여러 전표를 하나의 결재로 올리고 싶습니다.

해결방법: 합산전표 생성 기능(ZTRRE0150)을 사용하세요.

  • 동일한 날짜에 작성된 여러 전표를 하나로 합산 가능
  • 합산 후 하나의 BPM 기안문으로 결재 요청
  • 결재 효율성 향상 및 업무 간소화 효과

❓ 외화 지급은 어떻게 처리하나요?

해결방법: 외화 지급 전용 프로세스를 따라주세요.

  • 지급통화를 USD, JPY 등으로 변경
  • 세금코드는 Z5 사용
  • 지급조건은 9000번 사용
  • 매월 말 지급처리됩니다

❓ 세금코드를 어떻게 선택해야 하나요?

  • 1단계: SAP가 설치되어 있는지 확인
  • 2단계: 연결 설정 확인

🎯 매출 전표 처리 프로세스

매출 전표는 회사의 수익을 인식하고 고객에게 청구서를 발행하는 과정입니다.

  • 1단계: 고객처 등록 - 새로운 고객 정보를 시스템에 등록
  • 2단계: 수입전표 입력 - 매출 발생에 대한 전표 작성
  • 선택: ERP 청구서 수동 발행 - 필요시 고객에게 청구서 발송
💡 주요 특징
• 자동 세금계산서 발행
• 실시간 매출 인식
• 통합 고객 관리
• BPM 승인 워크플로우
